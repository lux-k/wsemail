Installation Steps

  1. Choose a name for your server. This name will be needed to obtain a certificate for your server.
  2. Obtain a certificate for your server. This step will vary depending upon what kind of CA infrastructure you choose to use. The following are two important items to note while creating your certificate.
    • The CN must be the name of your server.
    • The email address of the certificate must be postmaster@server.
  3. Run the certificate tool from the Program Group. You may choose to install the Penn Security Lab root, if you plan to communicate with our servers. The important button to use is the "Install Server Cert" button, which will install the certificate you created in step 2 in the correct place and set the permissions on the key correctly. The recommended format for loading the cert is a PKCS12 (pfx) bundle, with or without a password.
  4. After you have completed the certificate setup, run the Configurator to setup the configuration file the server will use to start. The on-screen instructions should guide you through setting up the database server, picking a name, etc.
  5. Test that the server is correctly setup. You can do this by going to http://localhost/WSEmailServer/. At the bottom of the page, a message will appear in either red of green informing you of the status of the server.
